Introduction

Propser is a peer-to-peer lending marketplace. Borrowers apply online for a fixed-rate, fixed-term loan between 2000 and $40,000 (here 1,000 and 35,000). Individuals and institutions can invest in the loans and earn attractive returns. Prosper handles all loan servicing on behalf of the matched borrowers and investors.

In this analysis I will focus on the question, which factors influence the borrowing rate an individual recieves when applying for a loan on Prosper.com. My exploration and analysis of the different variables in the dataset and their relationship are the basis for a linear model, I introduce in the second part, which can predict the borrowing rate an indivudal will recieve.

A detailed description of all the variables in the dataset can be found here.

Data Wrangling & Cleaning

Looking at the structure of the dataset, we can notice a couple of things we need to fix in order to analyze the data more easily:

  • According to the Prosper API documentation, the ListingCategory..numeric. variable describes the Listing category the borrower selected when posting their listing. We want to replace the numeric value with the category belonging to this number.
  • Secondly, we need to adjust the structure of the LoanOriginationDate and the LoanOriginationQuarter columns to be able to analyze the data over time. The lubridate, dplyr and tidyrpackages come in handy here.
  • And third, we combine two variables of creditworthyness, the CreditGrade (used for loans pre 2009) and the ProsperRating(alpha)(used for loans originated after July 2009) into one column called CreditRating.
# Replacing the numeric listing category with the category defined by the API
code2category <- function(code) {
  categories = c("Not Available", "Debt Consolidation", "Home Improvement",
                 "Business", "Personal Loan", "Student Use", "Auto", "Other",
                 "Baby&Adoption", "Boat", "Cosmetic Procedure", 
                 "Engagement Ring", "Green Loans", "Household Expenses",
                 "Large Purchases", "Medical/Dental", "Motorcycle", "RV",
                 "Taxes", "Vacation", "Wedding Loans")
  categories[2+1]
  return (categories[(code + 1)]) # in R Vector index starts at 1 not 0
}
prosper$ListingCategory <- sapply(prosper$ListingCategory, code2category)
# Parsing the LoanOriginationDate column into the "date"-Format
prosper$LoanOriginationDate <- as_date(prosper$LoanOriginationDate)


# Transforming LoanOriginationQuarter from [Q# Year] to [Year Q#]
# This will make sorting and plotting much easier
prosper$LoanOriginationQuarter <- as.character(prosper$LoanOriginationQuarter)
prosper <- prosper %>%
            separate(col = LoanOriginationQuarter,
                      into = c("Quarter", "Year"), sep = " ") %>%
            unite(col = LoanOriginationQuarter, Year, Quarter, sep = " ")

# Making a LoanOriginationQuarterF column
prosper$LoanOriginationQuarterF <- factor(prosper$LoanOriginationQuarter)

# Due to the transformed LoanOriginationQuarter column the standard order is
# Q1 2006 Q2 2006 Q3 2006 Q4 2007 Q1 2007...
# Joining the ProsperScore(alpha) and the CreditGrade into one column
prosper <- unite(prosper, col = "CreditRating", 
                 c("CreditGrade", "ProsperRating..Alpha."), sep = "")
prosper$CreditRating <- as.character(prosper$CreditRating)

# Setting CreditRating with the values "" or NC to NA
prosper$CreditRating[prosper$CreditRating == ""] <- NA
prosper$CreditRating[prosper$CreditRating == "NC"] <- NA

# Turning the variable into an ordered factor
prosper$CreditRating <- ordered(prosper$CreditRating, 
                              labels = c("AA","A","B","C","D", "E", "HR"))

Building a linear model to predict the Borrower Rate

# Building the model, bottom up approach
m1 <- lm(BorrowerRate ~ CreditRating, data = prosper)
m2 <- update(m1, ~ . + StatedMonthlyIncome)
m3 <- update(m2, ~ . + IsBorrowerHomeowner)
m4 <- update(m3, ~ . + EmploymentStatus)
m5 <- update(m4, ~ . + AvailableBankcardCredit)
m6 <- update(m5, ~ . + DebtToIncomeRatio)

# Summary of the r-squared tests of the different models

This is pretty good. (A r-square value of 1 would mean that our fitted regression line would explain 100% of the datapoints.)

By exploring and analyzing the dataset of Propser, I have been able to gain insights about the P2P lending business in general and also the business development of Prosper.com. One of the main metrics of interest for loan applicants and lenders is the borrowing rate. I was able to uncover the key factors that influence the interest rate an individual recieves. Furthermore I was able to build a linear model to predict the borrowing rate one recieves, taking into account the credit rate, income, employment status, home-ownership and credit and debt info of an individual.

One of the main challenges was to get familiar with the different variables in the data. With 81 variables per record, exploratory data analysis and plots are particulary important to gain an understanding of the data and to uncover relationships between different variables. R has proven to be a very effective. here.

A promising avenue of future research on the dataset is the analysis of factors which are signs that a loan will not be payed back fully or on time. Building a model which could predict the likelyhood of delinquent loans would greatly reduce the risk taken by the lender and would be a valuable tool for businesses such as Prosper.com.
